## 一、项目总览

WESSystemQD 是一套面向仓储管理领域的 **AI-WMS PLUS** 系统，采用 **前后端分离架构**，由两个核心项目组成：

| 项目 | 类型 | 技术栈 | 端口 | 职责 |
|------|------|--------|------|------|
| **WmsPlus** | 前端 | Blazor WebAssembly + AntDesign | 5192 | 用户界面、交互、状态管理 |
| **WmsPlus.Api** | 后端 | ASP.NET Core Web API + EF Core | 5102 | 业务逻辑、数据访问、认证授权 |

### 顶层目录结构

```
e:\AICode\WESSystemQD\
├── WmsPlus/                    # 前端项目（Blazor WASM）
├── WmsPlus.Api/                # 后端项目（ASP.NET Core API）
```

### 2.6 认证机制

**核心服务文件**（位于 `WmsPlus/Services/`）：

| 文件 | 作用 |
|------|------|
| `ApiConfig.cs` | 静态类，存储 API 基地址（默认 `http://localhost:5102`） |
| `TokenProvider.cs` | 轻量级 Token 持有者（Scoped），打破循环依赖 |
| `AuthService.cs` | 认证服务：登录、登出、修改密码、Token 验证、localStorage 持久化 |
| `AuthHttpClient.cs` | 静态工具类，创建带 Bearer Token 的 HttpRequestMessage |

**认证流程**：

1. **登录**：`AuthService.LoginAsync()` POST 到 `/api/auth/login`，返回 JWT Token + UserInfo
2. **存储**：Token 和 User 信息存入 localStorage（`wms_token`、`wms_user`）
3. **请求认证**：所有 API 请求必须使用 `AuthHttpClient.CreateRequest(HttpMethod, url, TokenProvider.Token)` 创建带 `Authorization: Bearer {token}` 头的请求
4. **验证**：启动时调用 `/api/auth/check` 验证 Token 有效性

**关键代码模式**（来自 `WmsPlus/Services/AuthHttpClient.cs`）：

```csharp
public static HttpRequestMessage CreateRequest(HttpMethod method, string url, string? token)
{
    var request = new HttpRequestMessage(method, url);
    if (!string.IsNullOrEmpty(token))
    {
        request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", token);
    }
    return request;
}
```

**App.razor 启动认证检查**：

1. 初始化 `AuthService.InitializeAsync()` 从 localStorage 加载用户
2. 若已认证，调用 `AuthService.ValidateTokenAsync()` 验证 token（5 秒超时）
3. 未认证且非登录页时，重定向到 `<RedirectToLogin />`
4. 初始化超时 8 秒保护

### 2.7 标签页系统

**核心特性**：采用"组件保持存活"模式

- 标签页切换时**不销毁**组件，通过 CSS `display:none/block` 切换显隐
- 保留表单输入数据，避免重复请求
- 每个标签页在 `DashboardLayout.razor` 中用 `@if` 占位
- 支持左右箭头导航，通过 JS Interop 控制平滑滚动
- 标签页标题文字使用 CSS 约束（`min-width: 0; overflow: hidden; text-overflow: ellipsis`）确保长标题时关闭按钮不被挤出

### 3.2 数据库上下文 (DbContext)

后端配置了 **两个 DbContext**，分别连接不同数据库：

#### AppDbContext

- **文件**：`WmsPlus.Api/Data/AppDbContext.cs`
- **连接数据库**：`wmssystem`（系统管理库）
- **注册实体**：6 个 DbSet

| DbSet | 表名 | 说明 |
|-------|------|------|
| User | pswd | 用户/密码 |
| DictTab | DICT_TAB | 表名字典 |
| DictFld | DICT_FLD | 字段说明字典 |
| Role | ROLE | 角色定义 |
| FxPswd | FX_PSWD | 角色权限 |
| PswdRole | PSWD_ROLE | 角色所属用户 |

#### WarehouseDbContext

- **文件**：`WmsPlus.Api/Data/WarehouseDbContext.cs`
- **连接数据库**：`db_gz01`（业务数据库）
- **注册实体**：**121 个 DbSet**
- **配置方式**：Fluent API（在 `OnModelCreating` 中为每个实体配置 `ToTable`、`HasKey`、`Property`）

**主键配置模式**：

| 表类型 | 配置方式 |
|--------|---------|
| 单主键表（大多数） | `entity.HasKey(e => e.ID)` |
| 复合主键表（表身 Tf_ 类） | `entity.HasKey(e => new { e.MAIN_NO, e.ITM })` |
| 无主键视图/只读表 | `entity.HasNoKey()` + `entity.ToView("VIEW_NAME")` |

> **重要约束**：每个新增 DbSet 实体必须在 `OnModelCreating` 中显式配置 `HasKey()`，否则首次查询会抛出 `InvalidOperationException` 导致该 DbContext 下所有 API 500 崩溃。

### 3.6 认证中间件

**配置文件**：`WmsPlus.Api/Program.cs`

```csharp
builder.Services.AddAuthentication(options =>
{
    options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
    options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
})
.AddJwtBearer(options =>
{
    options.TokenValidationParameters = new TokenValidationParameters
    {
        ValidateIssuer = true,
        ValidateAudience = true,
        ValidateLifetime = true,
        ValidateIssuerSigningKey = true,
        ValidIssuer = jwtIssuer,
        ValidAudience = jwtAudience,
        IssuerSigningKey = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(jwtKey))
    };
});
```

**中间件管道顺序**（`Program.cs`）：

1. `app.UseStaticFiles()` - 静态文件服务
2. `app.UseCors("AllowBlazorApp")` - CORS 策略
3. `app.UseAuthentication()` - JWT 认证
4. `app.UseAuthorization()` - 授权
5. `app.MapControllers()` - 路由映射

**JWT Token 生成**（`AuthController.cs`）：

- 算法：HmacSha256
- Claims：用户名、CompNo、Usr、Dept
- 过期时间：480 分钟（可配置）

## 五、两个系统之间的联系

### 5.1 通信协议与方式

**通信协议**：HTTP REST API（JSON 格式）

**请求流程**：

1. 前端通过 `HttpClient` 发送请求
2. 使用 `AuthHttpClient.CreateRequest()` 创建带 Bearer Token 的请求
3. 后端通过 JWT 中间件验证 Token
4. 后端 Controller 处理请求，通过 EF Core 访问数据库
5. 返回 JSON 数据

### 5.2 端口配置

| 服务 | 端口 | 配置文件 |
|------|------|---------|
| 后端 API (WmsPlus.Api) | `5102` (HTTP) / `7079` (HTTPS) | `WmsPlus.Api/Properties/launchSettings.json` |
| 前端 Blazor (WmsPlus) | `5192` (HTTP) / `7224` (HTTPS) | `WmsPlus/Properties/launchSettings.json` |
| 前端 API 基地址 | `http://localhost:5102` | `WmsPlus/wwwroot/appsettings.json` 的 `ApiBaseUrl` |

**启动命令**：

- 后端：`dotnet run --urls "http://localhost:5102"`（在 WmsPlus.Api 目录）
- 前端：`dotnet run --urls "http://localhost:5192"`（在 WmsPlus 目录）

### 5.3 认证流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户登录                                                  │
│    前端 AuthService.LoginAsync()                            │
│    → POST /api/auth/login {username, password}              │
├─────────────────────────────────────────────────────────────┤
│ 2. 后端验证                                                 │
│    AuthController.Login()                                   │
│    → 查询 wmssystem.pswd 表验证用户                          │
│    → 生成 JWT Token (HmacSha256, 480 分钟过期)              │
│    → 返回 { token, userInfo }                               │
├─────────────────────────────────────────────────────────────┤
│ 3. 前端存储                                                 │
│    AuthService 存入 localStorage:                            │
│    - wms_token: JWT Token                                   │
│    - wms_user: 用户信息 JSON                                 │
│    TokenProvider.Token = token (内存)                       │
├─────────────────────────────────────────────────────────────┤
│ 4. 后续请求                                                 │
│    AuthHttpClient.CreateRequest()                           │
│    → 添加 Header: Authorization: Bearer {token}             │
│    → 后端 JwtBearer 中间件验证 Token                         │
│    → [Authorize] Controller 处理请求                         │
├─────────────────────────────────────────────────────────────┤
│ 5. 启动验证                                                 │
│    App.razor 初始化时                                       │
│    → AuthService.ValidateTokenAsync()                       │
│    → GET /api/auth/check (5 秒超时)                         │
│    → 失败则重定向到登录页                                     │
└─────────────────────────────────────────────────────────────┘
```

### 5.5 关键架构特点

1. **双数据库架构**：业务数据（db_gz01）与系统数据（wmssystem）分离，通过不同 DbContext 访问
2. **字段字典系统**：wmssystem.DICT_TAB + DICT_FLD 提供 389 张表、5508 个字段的中文名称映射，用于动态表单渲染
3. **组件保持存活**：前端标签页切换不销毁组件，通过 CSS display 控制显隐，保持状态
4. **统一认证**：所有 API（除登录）需 `[Authorize]`，前端通过 `AuthHttpClient` 统一注入 Token
5. **CORS 配置**：后端允许前端 5192/5193/5196 等端口跨域访问
6. **开发规范**：项目包含 8 个规则文件（`.trae/rules/`），强制要求 EF Core HasKey 配置、认证组件使用等

2. **启动后端 API**：
   ```bash

   ```

   ```bash
   cd WmsPlus
   dotnet run --urls "http://localhost:5192"
   ```
4. **访问系统**：浏览器打开 `http://localhost:5192`

### 冒烟测试检查点

每次后端代码修改（尤其是涉及 DbContext、实体类、Controller）后，必须验证以下 API 正常返回：

```
GET http://localhost:5102/api/warehouse/search?pageIndex=1&pageSize=3     → 应返回 JSON 数据
GET http://localhost:5102/api/warehouse/columns?tableName=MY_WH.DB        → 应返回字段列表
```

如果任一接口返回 500，说明 **WarehouseDbContext 模型有问题**，必须立即排查修复。
