### 2.1 触发入口
- 在表格工具栏 **"操作" 列的表头右侧** 新增一个 **栏位设置按钮**（齿轮图标 ⚙️ 或类似图标）

### 2.3 功能细节

| 功能点 | 说明 |
|--------|------|
| **显示开关** | Toggle 开关，控制该列在表格中是否可见 |
| **栏位原名** | 英文字段名，只读，不可编辑 |
| **栏位标题** | 中文标题，用户可自定义修改（如将"单据日期"改为"日期"） |
| **宽度** | 数字输入，单位 px，控制列宽（0 表示自适应） |
| **搜索栏** | 对话框顶部搜索框，模糊匹配"栏位原名"，实时过滤列表 |
| **全选** | 左下角，一键开启所有列的显示 |
| **恢复默认** | 左下角，重置为默认配置（显示/隐藏状态、标题、宽度） |
| **确定** | 应用设置并关闭对话框 |
| **取消** | 放弃修改并关闭对话框 |

### 2.4 配置持久化（完整闭环）
- 用户设置的栏位配置保存到 **localStorage**
- 键名：`wms_column_settings_inbound_notice`
- **保存内容包含全部可编辑项**：显示/隐藏状态、自定义栏位标题、宽度
- 下次打开页面自动加载上次配置 → 表格立即使用自定义标题渲染
- 再次打开栏位设计对话框 → 显示之前保存的所有修改（标题、显隐、宽度均保留原样）
- 提供"恢复默认"功能清除本地配置并重置为出厂设置

**用户操作流程示例**：
1. 打开栏位设计 → 将"单据日期"的栏位标题改为"日期" → 点击确定
2. 表格表头立即变为"日期"
3. 刷新页面 → 表格表头仍为"日期"（从 localStorage 恢复）
4. 再次点击栏位设置按钮 → 对话框中该字段仍显示"日期"（非默认值"单据日期"）

## 四、实施步骤

### Step 1：定义栏位配置数据结构
```csharp
// 栏位配置项
private class ColumnConfig
{
    public string FieldKey { get; set; }       // 字段标识（如 "ItemNo"）
    public string OriginalName { get; set; }    // 原始英文名
    public string DisplayName { get; set; }     // 显示的中文名称（可编辑）
    public bool Visible { get; set; }           // 是否显示
    public int Width { get; set; }              // 宽度(px)，0=默认
    public int DefaultWidth { get; set; }       // 默认宽度（用于恢复）
}
```

### Step 2：初始化默认栏位列表（15 个字段）
- 定义完整的 15 个字段的默认配置数组
- 包含每个字段的 FieldKey、OriginalName、DisplayName 默认值、Visible 默认值、DefaultWidth

### Step 3：工具栏新增栏位设置按钮
- 在操作列表头（`<th class="col-action">`）旁边或工具栏区域添加齿轮图标按钮
- 点击触发打开栏位设置对话框

### Step 4：实现栏位设置对话框
- 模态弹窗，包含搜索栏、列表、全选/恢复默认/确定/取消按钮
- 列表每行：次序、Toggle开关、栏位原名（只读）、栏位标题（可编辑）、宽度（数字输入）
- 搜索栏支持模糊过滤栏位原名

### Step 5：改造表格为动态渲染
- `<thead>` 和 `<tbody>` 不再硬编码列，改为根据 ColumnConfig.Visible 遍历生成
- 操作列始终固定在最右侧
- 列宽从 ColumnConfig.Width 动态设置 style
- 空数据行的 colspan 动态计算

### Step 6：配置持久化
- 打开页面时从 localStorage 加载配置 → 合并到默认配置
- 点击确定时保存到 localStorage
- 点击恢复默认时清除 localStorage 并重置

### Step 7：修复表头文字居中
- CSS 中 `.data-table th` 添加 `text-align: center`（或确认现有样式中已有）

## 五、注意事项

1. **操作列不参与栏位设置**：操作列固定显示，不出现在栏位设置列表中
2. **导出功能联动**：导出 Excel 时应导出所有可见列的数据（与表格一致），或导出全部字段 —— 建议导出全部字段以保持完整性
3. **colspan 动态计算**：空数据行的 colspan 需根据当前可见列数 + 操作列动态计算
4. **排序图标处理**：原表头中有 sortable 类的列带有排序图标，动态渲染时需保留此能力
